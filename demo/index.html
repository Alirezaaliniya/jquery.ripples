WebGL Ripples

A water ripple effect for background images, images and videos.

Give any region a class, drop an image or a video inside it, and the effect takes over the pixels — no need to duplicate the URL in your JavaScript.

1. A video region

The plugin finds the <video> inside the container, uploads every decoded frame to the GPU and ripples it. Move your pointer over it.

A <video> already in the markup — picked up automatically. 
<!-- markup -->
<div class="demo-region">
  <video src="loop.mp4" autoplay muted loop playsinline></video>
</div>

// script
Ripples.attach('.demo-region');        // or $('.demo-region').ripples()

2. Zero JavaScript — just an attribute

Anything carrying data-ripples is initialised automatically on page load. Options come from data-ripples-* attributes.

An <img> with data-ripples — no script at all. 
<div data-ripples
     data-ripples-resolution="256"
     data-ripples-drop-radius="14"
     data-ripples-perturbance="0.05">
  <img src="photo.jpg">
</div>

3. A video by URL, without any markup

Pass the URL and the plugin creates (and hides) the video element for you. This one is non-interactive and fed by automatic drops.

Source given as { source: 'video/loop.webm' }. 
Ripples.attach('#url-region', {
  source: 'video/loop.webm',
  fit: 'cover',
  interactive: false
});

4. The classic: a CSS background

The original behaviour is untouched — this page's <body> and this panel both use their background-image, honouring background-size, -position and -attachment.
